# Vendor Selection Brief — SSO & User Database
**Project:** CazéTV World Cup 2026 Predictor (Bolão)
**Date:** April 2026
**Status:** 🔴 Urgent — Critical blocker for LiveLike integration

---

## 1. Project Context

CazéTV is building a **digital prediction game (bolão) for the 2026 FIFA World Cup**, a fan engagement platform where users make match predictions, earn points, compete in leagues, and receive prizes.

The engagement platform will be built on top of **LiveLike**, a US-based SaaS solution specialized in fan engagement (clients include Yahoo Sports, Bleacher Report, PFL, and Chelsea FC). LiveLike manages all game logic — leaderboards, leagues, badges, AI Guess, and live predictions.

**LiveLike does not store identity data (PII).** It operates on a federated identity model: it receives an authentication token generated by our own identity system (SSO), creates an internal mapping (LiveLike ID ↔ CazéTV ID), and only stores gameplay data. **All user identity — registration, login, authentication — is CazéTV's responsibility.**

---

## 2. The Challenge

We need to hire a vendor who takes full ownership and responsibility for:

1. **Designing and implementing** CazéTV's digital identity system and user database
2. **Operating and maintaining** this infrastructure throughout the entire project lifecycle (World Cup 2026 and beyond)
3. **Guaranteeing SLA for availability and scalability**, especially during extreme traffic spikes

### Why this is complex

This is not a standard corporate SSO project (e.g., 100 internal employees). We are talking about:

- **Mass B2C audience** — Brazilian football fans with no technical background
- **Unpredictable registration spikes** — expectation of **millions of account creations within minutes** during the platform launch and game announcements
- **Aggressive delivery window** — the MVP must be live and integrated with LiveLike before the start of the 2026 World Cup
- **Future extensibility** — the architecture must support the eventual addition of a sponsor (potentially iFood) as a second identity provider, without full refactoring

## 3. Expected Vendor Scope

### 3.1 Implementation (Phase 1 — Critical)

| Deliverable | Description |
|---|---|
| **User database** | Scalable database to store CazéTV user identity, PII, and credentials |
| **Registration flow** | Account creation via email/password and social login (Google, Apple — TBD) |
| **Authentication & session** | Issuance of JWT/OIDC token compatible with LiveLike's expected standard |
| **SSO endpoint** | Authentication endpoint called by LiveLike to validate the user and receive the token |
| **LiveLike handshake** | Integration and certification of the full flow: CazéTV Auth → LiveLike ID mapping |
| **Peak traffic management** | Auto-scaling strategy and warm-up plan to support massive simultaneous account creation |

### 3.2 Ongoing Operations (Phase 2 — Sustained)

| Deliverable | Description |
|---|---|
| **24/7 monitoring** | System health dashboards with proactive alerting |
| **Uptime SLA** | Minimum 99.9% availability — especially during match days |
| **Incident response** | Maximum 30-min response time for P1 incidents (auth down = platform down) |
| **Security updates** | Patches and preventive maintenance included in the contract |
| **Full documentation** | All architecture documented and handed over to CazéTV (no lock-in) |

---

## 4. Minimum Technical Requirements

> The vendor is **not required** to use a specific solution (Auth0, Firebase, Cognito, or proprietary stack). What matters is the outcome and contractual accountability.

- Native support for **OIDC / OAuth 2.0 / JWT** — mandatory for LiveLike integration
- Architecture hosted on **public cloud** (AWS, GCP, or Azure) with auto-scaling configured
- Proven capacity to handle **peaks of 100k+ req/min** for authentication
- Compliance with **Brazil's LGPD** — PII stored in Brazil or with a proper data processing agreement
- Extensible architecture for **future identity federation** (e.g., iFood as a secondary IdP)
- Delivery of a **functional MVP within 15 days** from project kick-off

---

## 5. Out of Scope

To avoid ambiguity in proposals:

- ❌ Frontend or UI development for the bolão (LiveLike + CazéTV's responsibility)
- ❌ CRM, push notifications, or communication workflows (separate scope)
- ❌ Game business logic (predictions, leaderboards, leagues) — 100% LiveLike
- ❌ End-user support (L1) — operated by CazéTV
